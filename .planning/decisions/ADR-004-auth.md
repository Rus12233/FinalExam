# ADR-0004: 인증 방식 선택

- 상태: Accepted
- 날짜: 2026-05-31
- 결정자: Rus12233

## 배경

개인 일기 앱으로서 사용자 식별 및 데이터 보호 방식을 결정해야 한다. 앱의 핵심 가치는 간편한 감성 기록이며, 가입 절차가 진입 장벽이 될 수 있다. 현재 모든 데이터는 기기 로컬에 저장된다.

## 고려한 대안

### 대안 A: 인증 없음 (로컬 전용)
- 장점: 즉시 사용 가능, 개인정보 수집 불필요, 구현 복잡도 없음
- 단점: 기기 분실/교체 시 데이터 복구 불가, 멀티 기기 동기화 불가

### 대안 B: Google/SNS 소셜 로그인
- 장점: 계정 기반 클라우드 백업 가능, 멀티 기기 동기화 가능
- 단점: Firebase 등 백엔드 연동 필수, OAuth 구현 복잡성, 이 앱 MVP 범위 초과

### 대안 C: 이메일 + 비밀번호
- 장점: 플랫폼 독립적 계정 관리
- 단점: 서버 필요, 비밀번호 분실 처리 필요, 개인정보 관리 책임 증가

## 결정

대안 A 인증 없음(로컬 전용)을 선택한다.

## 이유

- 현재 MVP 목표는 감성 일기 기록 경험 제공이며 계정 시스템은 범위 외
- 인증 없이도 로컬 Hive 저장소로 데이터 영속성 보장
- 가입 절차 없이 앱 설치 즉시 사용 가능 → 사용자 진입 장벽 최소화
- 개인정보를 수집하지 않아 개인정보처리방침 의무 없음

## 결과 (예상되는 영향)

긍정:
- 설치 즉시 사용 가능한 간편한 UX
- 서버 비용 및 인증 관련 보안 취약점 없음

부정 / 제약:
- 기기 변경/초기화 시 일기 데이터 유실 가능
- 멀티 기기 동기화 불가
