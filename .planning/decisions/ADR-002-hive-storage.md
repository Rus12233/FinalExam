# ADR-0002: 로컬 저장소로 Hive 선택

- 상태: Accepted
- 날짜: 2026-05-31
- 결정자: Rus12233

## 배경

일기 데이터를 앱을 껐다 켜도 유지해야 하며, 서버 없이 완전 오프라인으로 동작해야 한다. 개인 일기 앱 특성상 데이터를 외부 서버로 전송하지 않는 것이 사용자 프라이버시 측면에서도 바람직하다.

## 고려한 대안

### 대안 A: Hive (`hive` + `hive_flutter`)
- 장점: 순수 Dart 기반 NoSQL로 빠른 읽기/쓰기, `TypeAdapter`로 타입 안전 직렬화, 설정 간단
- 단점: 복잡한 쿼리(날짜 범위 검색 등) 지원이 SQLite보다 약함, TypeAdapter 코드 수동 작성 필요

### 대안 B: SQLite (`sqflite`)
- 장점: 복잡한 쿼리 가능, 관계형 데이터 모델 지원, 검증된 안정성
- 단점: SQL 스키마 설계 오버헤드, 객체 ↔ 테이블 매핑 코드 필요, 이 앱 규모에는 과도함

### 대안 C: SharedPreferences
- 장점: 가장 단순한 키-값 저장소, 설정값 저장에 최적화
- 단점: 리스트/객체 직렬화 미지원, 일기 목록 저장에 근본적으로 부적합

## 결정

대안 A Hive를 선택한다.

## 이유

- 단순 객체 목록 저장에 특화되어 이 앱의 `DiaryEntry` 구조와 딱 맞음
- TypeAdapter를 통해 JSON 파싱 없이 타입 안전하게 직렬화 가능
- SQLite의 스키마 설계 오버헤드 없이 빠르게 구현 가능
- 오프라인 전용 앱이므로 클라우드 연동 솔루션 불필요

## 결과 (예상되는 영향)

긍정:
- 앱 종료 후 재시작 시 일기·식물 상태 완전 복원
- 네트워크 없이 모든 기능 동작

부정 / 제약:
- `diary_entry.g.dart` TypeAdapter 코드를 수동으로 유지해야 함
- 날짜 범위 검색 등 고급 쿼리 필요 시 앱 레이어에서 필터링해야 함

## 후속 작업

- [x] `hive`, `hive_flutter` 패키지 추가
- [x] `DiaryEntry` TypeAdapter 작성 (`diary_entry.g.dart`)
- [x] `main.dart`에서 Hive 초기화
- [ ] 데이터 마이그레이션 전략 수립 (스키마 변경 시)
