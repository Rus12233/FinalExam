# ADR-0001: 모바일 프레임워크 선택

- 상태: Accepted
- 날짜: 2026-05-31
- 결정자: Rus12233

## 배경

iOS와 Android를 모두 지원하는 감성 다이어리 앱을 개발해야 했다. 파티클 애니메이션, 부드러운 식물 성장 연출이 핵심 기능이므로 UI 렌더링 성능이 중요했고, 단일 코드베이스로 유지보수 부담을 줄여야 했다.

## 고려한 대안

### 대안 A: Flutter (Dart)
- 장점: 단일 코드베이스로 iOS/Android/Web 동시 지원, Skia/Impeller 기반 60fps 렌더링, `flutter_animate` 등 풍부한 UI 패키지
- 단점: Dart 언어 별도 학습 필요, 바이너리 크기 약 8MB 추가

### 대안 B: React Native (JavaScript/TypeScript)
- 장점: 웹 개발자 친화적, 생태계 넓음
- 단점: 네이티브 브리지 레이어로 복잡한 애니메이션 성능 저하 가능, 감성 커스텀 UI 구현 난이도 높음

### 대안 C: 네이티브 (Swift + Kotlin 각각)
- 장점: 최고의 성능, 플랫폼별 UX 최적화
- 단점: 코드베이스 2개 유지, 개발 기간 2배

## 결정

대안 A Flutter를 선택한다.

## 이유

- 단일 코드베이스로 iOS/Android/Web을 커버해 유지보수 비용 최소화
- Impeller 엔진으로 파티클·식물 성장 애니메이션을 60fps로 구현 가능
- `flutter_animate` 패키지로 선언형 애니메이션 작성 가능
- React Native 대비 네이티브 UI 의존도가 낮아 감성 커스텀 디자인에 유리

## 결과 (예상되는 영향)

긍정:
- 하나의 코드로 멀티플랫폼 배포 가능
- 애니메이션 품질이 앱의 감성 컨셉과 잘 맞음

부정 / 제약:
- Dart 학습 비용 발생
- 앱 바이너리 크기가 네이티브 대비 약 8MB 증가
