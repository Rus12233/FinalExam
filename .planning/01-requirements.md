## 기능 요구사항 (Functional Requirements)

### FR-01: 일기 작성
- 사용자는 제목과 본문을 입력해 일기를 저장할 수 있다
- 저장 시 고유 ID(UUID v4)와 작성 시각이 자동 생성된다
- 빈 제목 또는 빈 본문은 저장되지 않는다

### FR-02: 식물 성장
- 총 일기 수에 따라 식물이 5단계로 진화한다
- 앱을 재시작해도 식물 단계가 유지된다
- 일기를 삭제하면 단계가 낮아질 수 있다 (의도된 동작)

### FR-03: 물방울 파티클
- 일기 저장 직후 화면에 물방울 파티클이 1.5초간 표시된다

### FR-04: 일기 목록
- 사용자는 과거에 작성한 일기를 날짜 역순으로 조회할 수 있다

### FR-05: 일기 삭제
- 사용자는 특정 일기를 영구 삭제할 수 있다

## 비기능 요구사항 (Non-Functional Requirements)

### NFR-01: 성능
- 앱 콜드 스타트 → 홈 화면 표시: 2초 이내
- 일기 저장 → 식물 업데이트: 즉각적 (16ms 이내)

### NFR-02: 오프라인
- 모든 기능은 인터넷 없이 동작해야 한다
- 데이터는 기기 로컬에만 저장된다

### NFR-03: 플랫폼
- Android (API 21+), iOS (13+), Web (Chrome 최신) 지원
