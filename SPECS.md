# SPECS.md — 도메인 규칙

> 이 프로젝트에서만 적용되는 도메인 고유 규칙을 정의합니다.
> 기술 선택 이유는 `.planning/decisions/ADR-*.md`를 참고하세요.

## 식물 성장 규칙

### 단계 기준 (`lib/models/plant_state.dart`)

| 단계 | 상수명 | 총 일기 수 | 이모지 | 표시 이름 |
|------|--------|-----------|--------|----------|
| 씨앗 | `seed` | 0개 | 🌰 | 씨앗 |
| 새싹 | `sprout` | 1~4개 | 🌱 | 새싹 |
| 어린나무 | `sapling` | 5~9개 | 🌿 | 어린나무 |
| 나무 | `tree` | 10~19개 | 🌳 | 나무 |
| 숲 | `forest` | 20개 이상 | 🌲 | 숲 |

- 단계는 항상 `totalEntries`(전체 일기 수) 기준으로만 계산됩니다.
- 일기를 삭제하면 단계가 낮아질 수 있습니다 (의도된 동작).

### 물방울 파티클 규칙

- 일기 작성 완료 시 1.5초간 물방울 파티클 재생 후 자동 비활성화
- `DiaryProvider.showWaterEffect` 플래그로 제어
- 파티클은 `PlantWidget` 내부에서 렌더링

## 일기 데이터 규칙

### DiaryEntry 필드 제약

| 필드 | 타입 | 제약 |
|------|------|------|
| `id` | String | UUID v4, 불변 |
| `title` | String | 필수, 최대 100자 |
| `content` | String | 필수, 빈 문자열 허용 안 함 |
| `createdAt` | DateTime | 작성 시각, 불변 |

- 일기 수정 기능은 현재 범위 밖입니다.
- 삭제는 영구 삭제입니다 (복구 없음).

### 저장소 규칙

- Hive Box 이름: `'diary'` (하드코딩, 변경 금지)
- 키: `entry.id` (UUID)
- 날짜 역순으로 목록 표시

## UI/UX 규칙

- 배경 팔레트: 따뜻한 자연 계열 (#F5EDD3, 초록 계열)
- 폰트 감성: 아기자기하고 따뜻한 느낌 유지
- 화면 전환: `MaterialPageRoute`로 기본 전환 사용
- 식물 단계 업그레이드 시 사용자에게 시각적 피드백 제공

## 범위 밖 (Out of Scope)

- 클라우드 동기화
- 사진 첨부
- 일기 공유
- 알림/리마인더
